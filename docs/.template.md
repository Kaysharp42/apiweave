# APIWeave Documentation Template

*Copy this file, rename it, and fill in each section. Every doc under `docs/` must follow this 7-section structure.*

> **Section requirements**: H1, description, and main content are always required. Prerequisites, Troubleshooting, and Related are required for task and tutorial docs. Pure reference docs may set Prerequisites to `None`. Table of Contents is required only when the doc exceeds 100 lines.

## Prerequisites

None. This is a meta-template for writing docs.

## The 7 Required Sections

Every doc must include the following sections in this order:

1. **H1 title**: single `#`, doc purpose in plain words.
2. **Description**: 1-2 sentences stating what the doc covers and who it's for. Italicize or use a short paragraph.
3. **Prerequisites**: what the reader must have done first. Link prior docs or list tools/accounts. Use `None` only for reference docs.
4. **Table of Contents**: required only when the doc exceeds 100 lines. Omit for shorter docs.
5. **Main content**: H2 and H3 sections. Each section must include at least one concrete example: a code block, numbered steps, a worked scenario, or a copy-paste command. Conceptual paragraphs alone are not enough.
6. **Troubleshooting**: operational Q&A. Each entry follows the pattern "If X happens, do Y." Conceptual "why" questions go to the central FAQ at `operations/troubleshooting.md`.
7. **Related**: at least two cross-links to other docs in this project.

## Skeleton

Copy and adapt this layout. Replace `<angle-bracket>` placeholders with real content.

````markdown
# Doc Title

*One or two sentences stating what this doc covers and who it's for.*

## Prerequisites

- Prior doc or setup step A
- Prior doc or setup step B

## <Main Topic 1>

Step-by-step content with a concrete example:

```bash
example command here
```

### <Subtopic 1A>

More detail with another example.

## <Main Topic 2>

- Bullet point with example
- Another bullet

## Troubleshooting

- **If X happens**, check Y or do Z.
- **If A fails**, run B and inspect output.

## Related

- [Other Doc 1](other-doc-1.md)
- [Other Doc 2](other-doc-2.md)
```
````

## Troubleshooting

- **If a section feels empty**, the doc is not ready. Merge it with another section or cut it.
- **If the doc exceeds its size cap** (per task spec), split it into sub-docs and link them from Related.
- **If a "why" question keeps coming up**, add it to the central FAQ at `operations/troubleshooting.md`, not here.

## Related

- [Documentation Hub](README.md)
- [Navigation Guide](NAVIGATION.md)

## FORBIDDEN CONTENT

The following patterns are not allowed in any file under `docs/`:

- **No `file:line` internal references** (e.g., `main.py:80-113`). Cite the file name only, or link to source on GitHub.
- **No TODO, TBD, coming-soon, or lorem-ipsum placeholders.** If content is missing, omit the section instead.
- **No marketing copy or "we believe" filler.** State what the product does, not what it aspires to be.
- **No actual unicode emoji glyphs.** Shortcodes like `:rocket:` are fine inside markdown renderers, but raw emoji characters in source are forbidden.
- **No screenshot or image references.** These docs are plain Markdown with no asset infrastructure. Use code blocks instead.
